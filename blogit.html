Creating Static Blogs with Blogit

Blogit is a simple git-based static blog generator. It automatically builds an Atom and a RSS feed and uploads static pages to your server using rsync.

Installing blogit

Clone the blogit repository using git. 

git clone https://pedantic.software/git/blogit

Copying Files

Enter the blogit directory. 

cd blogit

Either copy the Makefile (blogit) to your local binary folder (e.g. ~/.local/bin), or to /usr/bin/ 

cp blogit ~/.local/bin/

Copy the man page and update the mandb (run these commands as root): 

cp blogit.1 /usr/share/man/man1
mandb

Now you are able to use blogit's manual page (man blogit).

Creating a Blog

Create a new directory for your blog. 

mkdir blog && cd blog

Initialize blogit. The program will request for a remote path. Type something like this: root@example.org:/var/www/example.org 

blogit init

Blogit should initialize a git repository automatically. If it did not, do it manually. 

git init

Syntax

Here is an example article:

Title
![alt](path/to/an/image.png)
    
Some plain text.
    
;It is a comment (does not appear in a generated blog).
    
$ENVIRONMENT_VARIABLE
    
*italic*
**bold**
    
- unordered list 1
- unordered list 2
- unordered list 3
    
1. ordered list 1
2. ordered list 2
3. ordered list 3
    
#H1
##H2
###H3
    
```
echo "this is a code sample"
```
    
[it is a link](http://example.org)
    
;tags: tag1 tag2 tag3

Structure

The following variables are always defined:

  • TITLE, the title of the article
  • DATE_POSTED, the first publication date
  • DATE_EDITED, the last edit (commit) date
  • TAGS, the tags parsed from "tags:" comments

The index page is built using the following templates (you can find them in the "templates" folder):

  • header.html
  • index_header.html
  • tag_list_header.html
  • tag_entry.html, for each tag
  • tag_separator.html, between each tag
  • tag_list_footer.html
  • article_list_header.html
  • article_entry.html, for each article entry
  • article_separator.html, between each article
  • article_list_footer.html
  • index_footer.html
  • footer.html

Article pages:

  • header.html
  • index_header.html
  • (formatted article file)
  • footer.html

You can edit these files as you wish.

Building Static Pages

First of all, stage and commit the changes you have made. 

git add .
git commit -m "Add an article."

Now build pages. 

blogit build

After that you will see a "blog" directory. It contains the static pages.

If you want to delete the static pages, use blogit clean.

Deploying Static Pages on a Server

Make sure you have rsync installed on your server and client. If you have not: 

apt install rsync

Blogit will copy the static pages to your server (which you can change in the "config" file). Issue following in the directory with your blogit project: 

blogit deploy

Now you can serve your pages using a web server (e.g. Nginx).
