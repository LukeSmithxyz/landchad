Customize Error Pages

Aims

This tutorial will show you how to display a custom webpage for any error responses your website's visitors might find.

Background

HTTP Response Status Codes simply tell us if a request to our webpage was successful. There are two types of error responses:

  1. client
  2. server

If you've tried out server side scripting, you may have ran into a server side response error: 502 Bad Gateway.

Your website's visitors are more likely to encounter a client response error. Anyone who has used the Internet before has seen a 404 Not found page. In fact, here's one for this website!

Why change the default?

As a Land Chad, you should have total control over how exceptions and errors are presented to your audience. You can make the error page funny, straight forward or anything else you can imagine.

Let's say your site has no styling (which it most likely doesn't). The default error message in black text on a white background would look fine. However, if a visitor winds up on this 404 page, then it would be helpful to provide a link back to your homepage.

Customizing error pages gives visitors more options while making your website's aesthetic consistent.

Nginx Configuration

To begin, use your favorite workflow to create a new webpage, in this case we will choose the most common error code and call it, 404.html We will keep this file in the root of our web directory for simplicity's sake.

Now, we must go to our website's configuration file: /etc/nginx/sites-enabled/landchad. Open this file with your favorite text editor and add the following line inside the server directive: 

server {
    ...
    error_page 404 /404.html ;
    ...
}

You can replace 404, with any valid error code.

Save and close the file. Then reload nginx: systemctl reload nginx. To test if everything's working, search for an invalid page on your website in the url bar.

You should see your new customized 404 error page!

Other error pages will display with their corresponding errors. For example, accessing a file or folder with incorrect permissions will throw a 403 Forbidden error. Not providing a password will return a 401 Unauthorized error.
