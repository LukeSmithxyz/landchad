Understanding Internet Protocol

Internet Protocol (IP) is how computers and servers comunicate with one another. It is the set of rules our computers use to make sure our data gets to where it needs to be. We will be going over three concepts we use to talk about how two computers might communicate.

  • Packets
  • IP Addresses
  • Ports

Everything from DNS to Firewalls is set up in terms of Internet Protocol. Understanding this can help you troubleshoot your servers and configure other settings to better fit your needs.

Packets

Packets are small chunks of data that are sent between computers. You can think of them as letters between computers. You have an envelope with the address it came from and the address its going toand the data is inside. However, unlike letters, computers typically need to send multiple packets to relay the whole message they want to send. For example it would work something like this.

Bob sends Charlie a message that says "Hi! How are you? Bob's computer might send a packet for each word. Then Charlie's computer would reassemble the message for him to see

There are multiple different kinds of packets, and they are usually made up of slightly different information based on their type. But, all packets include a destination, where it came from, and the data that is being sent.

IP Addresses

So next let's learn how to write the address that packets use. They are called IP addresses or IP's for short. An IP is made of four numbers ranging from 0 to 255. Each of them are separated by periods. Each of these numbers are held in 1 byte, and those four bytes make up the IP address. Each computer on the internet has at least one of these addresses. That includes servers for web pages. Another way you could access a webpage is by typing the IP address into the address bar of your browser. For example 142.250.138.100 is one of the IP's that leads to google.com. If you type that in, it should take you straight to google.

Ports

So one last thing to discuss is ports. Each time you connect to another computer or to a website, you have to use a specific port at their specific IP Address. A port is kind of like a window or door into the computer. Some of these are open. Some are locked. Either way the only way in is through a port. These ports are indicated by a number. For instance, port 80 is usually used when connecting to a webpage.

You can write an IP Address/Port pair like this IPAddress:Port. If you wanted to connect to google.com on port 80, you could write it 142.250.138.100:80.
