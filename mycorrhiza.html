Mycorrhiza wiki

Mycorrhiza wiki is a free as in freedom wiki engine. It's tailored toward personal wikis and wikis for small teams or communities. For larger wikis, consider MediaWiki. Like most of the tutorials on LandChad.net, this tutorial involves Debian and nginx.

Whilst Mycorrhiza is running, the terminal cannot be used for other commands. To resolve this, we need to install tmux, a terminal multiplexer, so that Mycorrhiza can run in a terminal in the background. git is also required for Mycorrhiza to function:

apt install git tmux

Make a folder for the wiki:

mkdir /var/www/wiki

Download the latest release of Mycorrhiza to your newly created wiki folder. Then untar it:

wget https://github.com/bouncepaw/mycorrhiza/releases/download/v1.8.2/mycorrhiza-v1.8.2-linux-amd64.tar.gz -O /var/www/wiki

tar -xf /var/www/wiki/mycorrhiza-v1.8.2-linux-amd64.tar.gz

We now need to create an nginx configuration file:

nano /etc/nginx/sites-available/wiki

Populate your file with the following:

server {
	server_name wiki.example.org www.wiki.example.org ;
	location / {
		proxy_pass http://127.0.0.1:1737 ;
	}
}

The URL of the wiki will be wiki.example.org, but you can change this to any sub-domain/domain.
1737 is the port that the Mycorrhiza devs chose. It can be changed in the config.ini file we will edit later.

Enable the nginx configuration by making a link to it in the sites-enabled directory:

ln -s /etc/nginx/sites-available/wiki /etc/nginx/sites-enabled

Add HTTPS authentication for the sub-domain (wiki.example.org). During the certbot procedure, you will need to enter e in order to expand your certificates.

certbot --nginx

Reload nginx for the nginx configuration file to come into effect:

systemctl reload nginx

We now need to run Mycorrhiza in a terminal in the background. For this step we will use tmux, a terminal multiplexer. The initial run of mycorrhiza will populate your chosen folder with some files, including a configuration file which can edit. Start by running tmux and creating a new window (terminal instance) by pressing Ctrl + B and then C:

tmux

then Ctrl + B and then C.

Now run mycorrhiza in that new window:

./var/www/wiki/mycorrhiza-v1.8.2-linux-amd64/mycorrhiza /var/www/wiki

Detach from the tmux session by pressing Ctrl + B and then D. It's a good idea to edit the wiki's configuration file (config.ini) in order to prevent anyone from modifying the wiki. In this file you can also change the aforementioned 1737 listen address if you so wish.

nano /var/www/wiki/config.ini

then:

...
[Network]
ListenAddr = 127.0.0.1:1737
...
[Authorization]
UseAuth = true
...

Your wiki will now be served at wiki.example.org. You should familiarise yourself with the other settings in config.ini. A guide for them is provided here.

exit
